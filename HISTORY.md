# 📜 观看历史/进度 - 完整功能指南

## 🎯 功能总览

观看历史功能自动记录你的观看进度，支持断点续播、统计分析等。

### 核心功能

| 功能 | 说明 | 状态 |
|------|------|------|
| ⏱️ 自动记录 | 观看时自动保存进度 | ✅ |
| 🔄 断点续播 | 从上次位置继续观看 | ✅ |
| 📊 进度显示 | 百分比进度条 | ✅ |
| 🔢 观看次数 | 统计每个视频观看次数 | ✅ |
| 📈 统计分析 | 总时长、完成率等 | ✅ |
| 📱 设备记录 | 记录观看设备 | ✅ |
| 🔍 筛选 | 全部/未完成/已完成 | ✅ |
| 🗑️ 管理 | 清空/单条删除 | ✅ |
| 💡 智能提示 | 继续观看提示 | ✅ |

---

## 🚀 使用指南

### 自动记录进度

**无需任何操作！**

1. 开始观看视频
2. 系统每 10 秒自动保存一次进度
3. 关闭播放器时保存最终进度
4. 下次观看可从上次位置继续

### 断点续播

**方法 1: 从历史记录**
1. 点击顶部 📜 按钮
2. 找到想继续的视频
3. 点击历史记录项
4. 自动从上次位置播放

**方法 2: 从视频列表**
1. 点击视频卡片
2. 如果有观看进度，弹出提示
3. 选择"继续"或"从头开始"

### 查看观看历史

1. 点击顶部导航栏的 📜 按钮
2. 查看所有观看记录
3. 每条记录显示:
   - 📹 视频文件名
   - 📊 进度百分比
   - ⏱️ 观看时长
   - 📅 最后观看时间
   - 🔢 观看次数

### 删除单条记录

1. 打开历史记录

**⚠️ 注意：** 清空后无法恢复！

## 🔌 API 接口

### 基础操作

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/history` | GET | 获取所有历史记录 |
| `/api/history/<filename>` | GET | 获取单个视频进度 |
| `/api/history/<filename>` | POST | 更新观看进度 |
| `/api/history/<filename>` | DELETE | 删除单条记录 |

### 高级功能

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/history/clear` | POST | 清空所有历史 |
| `/api/history/recent` | GET | 获取最近记录 |
| `/api/history/stats` | GET | 获取统计数据 |

### 使用示例

**更新进度:**
```javascript
fetch('/api/history/video.mp4', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
        position: 120,  // 秒
        duration: 300,  // 秒
        device: 'web'
    })
})
```

**获取进度:**
```javascript
fetch('/api/history/video.mp4')
    .then(r => r.json())
    .then(data => {
        console.log(`进度：${data.progress_percent}%`);
        console.log(`位置：${data.position}秒`);
    });
```

**获取统计:**
```javascript
fetch('/api/history/stats')
    .then(r => r.json())
    .then(stats => {
        console.log(`观看时长：${stats.total_watch_time_hours}小时`);
        console.log(`已完成：${stats.completed_videos}个`);
    });
```

## 💾 数据存储

### 存储位置

```
video_site_project/
└── watch_history.json    # 观看历史数据
```

### 数据格式

```json
{
  "video.mp4": {
    "position": 120,
    "duration": 300,
    "last_watched": "2026-03-25 19:05:58",
    "progress_percent": 40.0,
    "watch_count": 3,
    "device": "web",
    "first_watched": "2026-03-25 18:00:00"
  }
}
```

### 字段说明

| 字段 | 说明 |
|------|------|
| position | 当前播放位置（秒） |
| duration | 视频总时长（秒） |
| last_watched | 最后观看时间 |
| progress_percent | 进度百分比 |
| watch_count | 观看次数 |
| device | 观看设备 |
| first_watched | 首次观看时间 |

## 🎯 最佳实践

### 进度保存策略

1. **定期保存**: 每 10 秒自动保存
2. **关闭保存**: 关闭播放器时保存
3. **阈值保护**: <10 秒不保存（避免误操作）
4. **完成标记**: ≥95% 标记为已完成

### 性能优化

1. **懒加载**: 历史记录分页加载
2. **缓存**: 统计数据缓存
3. **异步**: 后台保存不阻塞
4. **清理**: 定期清理旧记录

### 隐私保护

1. **本地存储**: 数据仅存储在本地
2. **可选清空**: 随时清空历史记录
3. **单条删除**: 可选择性删除
4. **无追踪**: 不上传服务器

## 🐛 常见问题

### Q: 为什么进度没有保存？
A: 观看时间<10 秒不会保存，这是为了避免误操作。

### Q: 如何从头开始播放？
A: 在继续观看提示中选择"取消"，或手动拖动进度条到开头。

### Q: 观看次数如何计算？
A: 每次打开视频播放算一次，重复观看会累加。

### Q: 完成标准是什么？
A: 观看进度≥95% 即标记为已完成。

### Q: 可以导出历史记录吗？
A: 目前不支持，可以直接编辑 `watch_history.json` 文件。

### Q: 多设备进度能同步吗？
A: 目前不支持，计划在未来版本添加云同步功能。
